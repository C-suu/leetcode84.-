### （1）题目意思与生僻概念解释

**题目意思：**
给出一排宽度均为 1 的相邻柱子，每个柱子有不同的高度。要求在这些柱子所组成的轮廓内部，画出一个最大的实心长方形，并求出这个长方形的面积。

**核心生僻概念解释：**

  * **单调栈（Monotonic Stack）：** 是一种特殊的栈数据结构。它在入栈和出栈的过程中，严格维持栈内元素的单调性（从底到顶单调递增或单调递减）。本题使用的是**单调递增栈**，即压入的新元素如果比栈顶元素小，就必须强行把栈顶较大的元素弹出，直到栈内重新满足递增为止。
  * **哨兵（Sentinel）：** 是一种编程技巧，通过在数据的两端添加无意义的“虚拟元素”（如本题中的 0 和 -1），来避免在循环中反复判断数组是否越界或栈是否为空，从而极大地精简代码逻辑。

### （2）代码解题思路解析

要找到最大的矩形，核心思想是：**枚举“高度瓶颈”**。
任意一个矩形，它的高度必然等于包含在它内部的最矮的那根柱子的高度。因此，只需要遍历每一根柱子，假设“以当前柱子的高度作为矩形的最终高度”，然后向左、向右尽可能延伸，算出面积。最后比较所有柱子作为“高度瓶颈”时算出的面积，取最大值即可。

**如何利用单调递增栈快速找到左右边界？**

  * **入栈规则：** 当遇到的柱子越来越高时，它们无法确定彼此的右边界（还能继续向右延伸），因此将它们的索引依次入栈。
  * **触发出栈（发现右边界）：** 当突然遇到一根较矮的柱子（当前高度 `v`）时，说明栈顶那根较高柱子的“右边界”被卡住了（无法再向右延伸）。此时，将栈顶柱子弹出并进行面积计算。
  * **计算左右边界：**
      * **高度：** 刚刚弹出的柱子的高度。
      * **右边界：** 就是当前触发弹出动作的较矮柱子的索引 `i`。
      * **左边界：** 弹出该柱子后，此时栈顶剩下的新元素，就是这根柱子左侧第一个比它矮的柱子的索引。
      * **宽度：** `右边界 - 左边界 - 1`。

代码中最精妙的一笔在于哨兵机制：

1.  **末尾追加高度 0：** 强制最后所有的柱子都能遇到一个“最低点”，从而依次出栈并结算所有未计算的面积。
2.  **栈底塞入索引 -1：** 配合 Python 的负索引特性，`heights[-1]` 恰好指向末尾追加的那个 `0`。这个高度为 0 的哨兵永远不可能比正数高度大，因此 `-1` 永远不会被弹出栈，完美避免了“栈为空”的边界报错，同时在计算宽度时提供了完美的左侧物理边界。

### （3）带有注释的代码框

```python
class Solution:
    def largestRectangleArea(self, heights: List[int]) -> int:
        # 初始化栈，放入索引 -1 作为左边界哨兵
        stack = [-1]
        # 在原数组末尾追加高度 0，作为右边界哨兵，迫使循环结束时清空栈内元素
        heights.append(0)
        # 初始化最大面积记录变量
        mxarea = 0
        # 遍历高度数组，i 为索引，v 为当前柱子的高度
        for i, v in enumerate(heights):
            # 当栈顶索引对应的高度大于当前柱子高度时，说明栈顶柱子的右边界找到了
            # 注意：stack[-1] 是栈顶索引，heights[stack[-1]] 是栈顶柱子的高度
            while heights[stack[-1]] > v:
                # 弹出栈顶索引，作为当前要计算矩形面积的“高度瓶颈”柱子
                mid = stack.pop()
                # 计算面积：高度为 heights[mid]
                # 宽度为 当前索引 i 减去 新的栈顶索引 (左边界) 再减 1
                area = heights[mid] * (i - 1 - stack[-1])
                # 如果当前计算出的面积大于历史最大面积，则更新
                if area > mxarea: 
                    mxarea = area
            # 上面的判断条件不如直接修改为：max_area = max(area, max_area)
            # 无论是否发生出栈，当前柱子的索引最终都要入栈，维持单调递增
            stack.append(i)
        # 遍历完毕，返回记录的最大面积
        return mxarea
```

主要难点在于“栈里存的是位置（索引），但比大小的时候要用高度”。

### 1. 概念通俗解释

假设要以某一根柱子的高度为基准，向右边画一个实心的长方形。
* 只要右边接下来的柱子**比这根基准柱子高（或一样高）**，这个长方形就可以畅通无阻地一直往右延伸。
* 但是，一旦遇到一根**比基准柱子还要矮**的柱子，这个长方形就“撞墙”了，被卡住了，绝对不可能再往右画了。

在这个逻辑下：
* **栈顶柱子（`heights[stack[-1]]`）：** 就是当前正在试图向右延伸的那根“基准柱子”。
* **当前柱子（`v`）：** 就是向右探索时，刚刚遇到的这根“新柱子”。
* **右边界找到了：** 当发现“新柱子”比“基准柱子”矮的时候（即 `heights[stack[-1]] > v`），说明“基准柱子”向右延伸的潜力耗尽了。这根矮矮的新柱子所在的位置，就是基准柱子无法跨越的**右边界**。

**为什么要存索引 `stack[-1]` 而不是直接存高度？**
因为最后计算面积的公式是 `宽度 = 右边界 - 左边界 - 1`。只有保存了柱子的**索引位置**，才能在撞墙时，算出物理上的宽度距离。有了索引，随时可以通过 `heights[索引]` 查出高度；但如果只存高度，就再也找不回它在原数组中的位置了。

### 2. 数值算例与变量剖析

设定一个极简的局部数组：`heights = [5, 6, 2]`。

**步骤 1：处理高度 5**
* 当前索引 `i = 0`，当前高度 `v = 5`。
* 栈是空的，直接把索引 `0` 放进去。
* **当前栈：`[0]`**

**步骤 2：处理高度 6**
* 当前索引 `i = 1`，当前高度 `v = 6`。
* 看一眼栈顶：栈顶存的索引是 `0`，对应的高度 `heights[0]` 是 5。
* 因为 5 没有大于 6（新柱子更高，不挡路），矩形还能继续往右画。不触发条件。
* 把高度 6 的索引 `1` 放进栈。
* **当前栈：`[0, 1]`**

**步骤 3：处理高度 2（关键触发点！）**
* 当前索引 `i = 2`，当前高度 `v = 2`。
* 看一眼栈顶准备进行判断，此时各个变量的映射关系如下：

| 代码变量 | 当前数值 | 物理含义说明 |
| :--- | :--- | :--- |
| `i` | **2** | 新遇到柱子的位置编号 |
| `v` | **2** | 新遇到柱子的绝对高度 |
| `stack[-1]` | **1** | 栈顶那根待处理柱子的位置编号 |
| `heights[stack[-1]]` | `heights[1]` = **6** | 栈顶那根待处理柱子的绝对高度 |

**判断逻辑：**
执行判断：`heights[stack[-1]] > v`  $\rightarrow$  即判断 `6 > 2`。
结论：**成立！**

**物理意义翻译：**
放在位置 `1`、高度为 `6` 的那根高柱子，本来满心欢喜地想把长方形往右延伸。结果它往右走了一步，发现位置 `2` 的柱子高度只有 `2`。高度为 `6` 的长方形直接被拦腰截断了，它永远过不去位置 `2`。

因此，代码在这个时刻认定：**对于位置 `1` 的这根柱子来说，它的绝对“右边界”就是当前这个矮柱子的位置 `i = 2`**。随后就可以安心地把它从栈里弹出来（`pop`），计算它能构成的最大面积了。

### （4）每一行代码详细中文解释

1.  `class Solution:`
      * **解释：** 定义名为 `Solution` 的类。
2.  `def largestRectangleArea(self, heights: List[int]) -> int:`
      * **解释：** 定义方法，接收整数列表 `heights` 表示各柱子高度，返回整数表示最大矩形面积。
3.  `stack = [-1]`
      * **解释：** 初始化单调栈，并在最底部放入一个虚拟索引 `-1`。由于后面在 `heights` 末尾加了 `0`，`heights[-1]` 的值将是 `0`。这就充当了一个永远不会被弹出的垫底哨兵，同时为最左侧的柱子提供计算宽度的边界。
4.  `heights.append(0)`
      * **解释：** 在原高度数组的最右侧硬塞入一根高度为 0 的虚拟柱子。这样当循环走到最后时，0 会小于前面所有正常的柱子，从而触发 `while` 循环，把栈里面残留的阶梯状上升的柱子全部结算掉。
5.  `mxarea = 0`
      * **解释：** 初始化全局最大面积变量为 0。
6.  `for i, v in enumerate(heights):`
      * **解释：** 使用 `enumerate` 函数遍历处理过的高度数组。`i` 会逐一获取每个元素的下标索引，`v` 会逐一获取对应的柱子高度。
7.  `while heights[stack[-1]] > v:`
      * **解释：** 开启循环判断，检查此时栈顶所保存的那个下标对应的高度（`heights[stack[-1]]`），是不是大于当前遍历到的柱子高度 `v`。如果是，说明破坏了单调递增性，栈顶柱子的“向右延伸能力”到头了。
8.  `mid = stack.pop()`
      * **解释：** 将由于遇到较矮柱子而被卡住的那个栈顶索引弹出来，赋值给 `mid`。以这个位置的柱子高度作为基准来计算面积。
9.  `area = heights[mid] * (i - 1 - stack[-1])`
      * **解释：** 核心面积公式。矩形的高是刚刚弹出的 `heights[mid]`。此时栈顶剩下的元素 `stack[-1]` 是由于前面弹出而暴露出来的“左侧比它矮的柱子”。右侧比它矮的柱子正是当前遍历的 `i`。因此夹在中间能画出的最大宽度为 `i - stack[-1] - 1`。两者相乘得到当前矩形面积。
10. `if area > mxarea: mxarea = area`
      * **解释：** 将刚刚算出来的 `area` 与之前保存的最大面积 `mxarea` 进行比较。如果打破了记录，就把新记录保存下来。
11. `stack.append(i)`
      * **解释：** 当 `while` 循环结束（此时栈顶柱子的高度要么小于要么等于当前高度 `v`，单调性恢复），把当前柱子的下标 `i` 压入栈中，等待后续寻找它的右边界。
12. `return mxarea`
      * **解释：** 当整个 `for` 循环结束，所有的计算完成，将历史最高记录 `mxarea` 作为函数的最终结果返回。
